## Prompt

/sp.constitution # Project Constitution: Evolution of Todo

## Overview
This constitution governs the development of the "Evolution of Todo" project for Hackathon II. It enforces Spec-Driven Development (SDD) using Claude Code and Spec-Kit Plus, shifting the role from syntax writer to system architect. The project evolves a Todo app from a simple CLI to a cloud-native AI system, incorporating Reusable Intelligence through skills and subagents.

## Core Principles
- **Spec-Driven Development (SDD)**: All features must start with a Markdown spec. Refine specs iteratively until Claude Code generates correct implementations. No manual coding is allowed‚Äîonly spec refinement and prompt engineering.
- **Agentic Dev Stack Workflow**: Follow Write spec ‚Üí Generate plan ‚Üí Break into tasks ‚Üí Implement via Claude Code. Use Spec-Kit Plus for spec management (e.g., init, generate plan, validate).
- **Reusable Intelligence**: Design modular skills and subagents using the P+Q+P framework (Problem + Query + Plan). For example, create reusable "TaskManagerSkill" for CRUD operations to earn bonus points.
- **Clean Code and Structure**: Adhere to Python best practices (PEP8), modular design, and proper project structure. Use UV for dependency management.
- **No Manual Interventions**: Code must be generated by Claude. Document all Claude sessions, prompts, and iterations in CLAUDE.md for judging.
- **Bonus Alignment**: Incorporate Reusable Intelligence (+200 points), Cloud-Native Blueprints (+200), Multi-language Support (Urdu in chatbot, +100), and Voice Commands (+200) where applicable across phases.
- **Ethical and Compliant Development**: Ensure user data isolation (e.g., via auth in later phases), scalability, and alignment with cloud-native principles.

## Phase-Specific Guidelines
### Phase I: In-Memory Python Console App
- Scope: Implement Basic Level features (Add Task, Delete Task, Update Task, View Task List, Mark as Complete) in a CLI app with in-memory storage.
- Data Structure: Use a list of dictionaries (e.g., {'id': int, 'title': str, 'description': str, 'complete': bool}).
- Interface: Menu-driven CLI with user prompts.
- Technology: Python 3.13+, UV, Claude Code, Spec-Kit Plus.
- Deliverables: Specs in /specs/ (with history), source in /src/, CLAUDE.md, README.md.

## Future Phases Alignment
- Evolve to full-stack (Phase II), AI chatbot (Phase III), local K8s (Phase IV), and advanced cloud (Phase V).
- Integrate authentication (Better Auth + JWT), databases (Neon PostgreSQL), agents (OpenAI Agents SDK, MCP SDK), and cloud tools (Docker, Minikube, Kafka, Dapr).
- Maintain monorepo structure, expanding to /frontend/ and /backend/ in later phases.

## Enforcement
All specs must reference this constitution. Violations (e.g., manual code) will invalidate submissions. Iterations must be tracked in /specs/history/ for transparency.
